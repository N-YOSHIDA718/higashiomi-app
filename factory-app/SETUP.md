# 工場管理システム セットアップガイド

## システム構成
- フロントエンド: Next.js → Vercelでホスティング（無料）
- バックエンド/DB: Google Apps Script + Googleスプレッドシート（無料）
- QRスキャン: スマホブラウザのカメラ（アプリ不要）

---

## ステップ1: Googleスプレッドシートの準備

1. [Google スプレッドシート](https://sheets.google.com) で新規スプレッドシートを作成
2. URLの中から **スプレッドシートID** をコピー
   ```
   https://docs.google.com/spreadsheets/d/【ここがID】/edit
   ```
3. このIDを控えておく

---

## ステップ2: Google Apps Script のセットアップ

1. スプレッドシートを開いた状態で `拡張機能 → Apps Script` をクリック
2. `gas/Code.gs` の内容を **すべてコピー** してエディタに貼り付ける
3. 以下の2行を自分の値に書き換える:
   ```javascript
   const SHEET_ID = '【ステップ1で控えたID】';
   const API_KEY  = '【任意の秘密キー（英数字推奨）】';
   ```
4. 保存ボタン（Ctrl+S）を押す
5. 関数リストから `setupSheets` を選んで「実行」→ シートが自動作成される
6. 「デプロイ」→「新しいデプロイ」をクリック
7. 種類: 「ウェブアプリ」/ アクセス: 「全員」 を選んで「デプロイ」
8. 表示される **デプロイURL** をコピーして控える

---

## ステップ3: ローカル開発（確認用）

```bash
# 依存関係インストール
npm install

# 環境変数を設定
cp .env.local.example .env.local
# .env.local を開いて以下を設定:
# NEXT_PUBLIC_GAS_URL=（ステップ2でコピーしたURL）
# GAS_API_KEY=（ステップ2で設定した秘密キー）

# 開発サーバー起動
npm run dev
# → http://localhost:3000 で確認
```

## ステップ4: Vercelへのデプロイ（本番公開）

### 方法A: Vercel CLI（推奨）
```bash
npm install -g vercel
vercel login
vercel

# 環境変数を設定
vercel env add NEXT_PUBLIC_GAS_URL
vercel env add GAS_API_KEY

# 本番デプロイ
vercel --prod
```

### 方法B: GitHub連携
1. このフォルダをGitHubにプッシュ
2. [Vercel](https://vercel.com) にログイン → 「New Project」
3. GitHubリポジトリを選択 → インポート
4. **Environment Variables** に以下を追加:
   - `NEXT_PUBLIC_GAS_URL` = デプロイURL
   - `GAS_API_KEY` = 秘密キー
5. 「Deploy」ボタンをクリック

デプロイ完了後、発行されたURLをスマホで開けば使えます。

## QRコードシールの準備

材料や図番にQRコードシールを貼ります。

### QRコード生成方法
1. Googleスプレッドシートの「材料在庫」シートを開く
2. QRコード列に以下のような式を入力:
   ```
   =IMAGE("https://chart.googleapis.com/chart?chs=200x200&cht=qr&chl="&ENCODEURL(A2))
   ```
   （A2 = 材料IDのセル）
3. 印刷してラミネート加工したシールを材料に貼付

## 各シートの列構成

### 図番マスタ
図番 / 品名 / 数量 / 材料種別 / 材料費 / レーザー費 / 曲げ費 / 溶接費 / タップ費 / 切断費 / その他加工費 / 検査費 / 事務費 / 原価合計 / 販売単価 / 粗利率 / 案件番号 / 納期 / 備考 / 更新日時

### 材料在庫
材料ID / 材料名 / 規格 / 単位 / 現在庫 / 単価 / 最小在庫 / 保管場所 / QRコード / 更新日時

### 入出庫履歴
日時 / 材料ID / 材料名 / 区分 / 数量 / 単位 / 前在庫 / 後在庫 / 図番 / 担当者 / 備考

### 工程進捗
図番 / 案件番号 / 工程 / 担当者 / 予定開始 / 予定終了 / 実績開始 / 実績終了 / ステータス / 備考 / QRコード / 更新日時

### ミス記録
日時 / 図番 / 案件番号 / 工程 / ミス種別 / 数量 / 損失金額 / 原因 / 対策 / 担当者

## 機能一覧

| 機能 | 説明 |
|------|------|
| 図番・原価管理 | 工程別（レーザー・曲げ・溶接など）に加工費を分けて管理。原価合計・粗利率を自動計算 |
| 材料在庫管理 | 入庫・出庫・端材入庫。在庫不足アラート。入出庫履歴を記録 |
| 工程進捗管理 | カンバンボード・一覧の2ビュー。担当者・予定/実績日付を管理 |
| ミス記録 | ミス種別・損失金額を記録。原価への影響（修正後粗利率）を自動計算 |
| QRスキャン | スマホカメラで材料・図番・工程QRを読み取り。入出庫・進捗更新をその場で実行 |

## よくある質問

**Q: GAS接続なしでも動きますか？**  
A: はい。GAS未接続の場合、デモ用のモックデータで動作します。
   実データを使うにはGASのセットアップが必要です。

**Q: 複数人で同時に使えますか？**  
A: はい。Vercelにデプロイ後、URLを共有すれば複数人が同時に使えます。
   データはGoogleスプレッドシートに保存されます。

**Q: スマホのQRスキャンはアプリが必要ですか？**  
A: いいえ。Safariまたはhromeのブラウザでそのまま使えます。
   カメラのアクセス許可が必要です。

**Q: バックアップは？**  
A: Googleスプレッドシートが自動でバージョン履歴を保存します。
